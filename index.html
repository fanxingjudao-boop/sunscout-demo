Satellite Vegetation Monitoring / for Solar O&M

現地に行く前に、
敷地の「緑」は見えている。

SUNSCOUT は、太陽光発電所の敷地を Sentinel-2 衛星で毎月見守り、 雑草・樹木の繁茂リスクを訪問前に検知する植生モニタリングサービスです。 実在の発電所で 24ヶ月分の実測を完了し、雲による欠測率まで定量化しています。

Problem

広い敷地、限られた人手。
植生の異変は、気づいた時には積み上がっている。

ISSUE — 01

数十haの敷地を、巡回では見きれない

メガソーラーの敷地は数十ヘクタール。フェンス際・法面・隣接林縁まで、目視巡回だけで植生の変化を追い続けるのは現実的ではありません。

ISSUE — 02

草刈りのタイミングが読めない

行ってみたら手遅れ、行ったけど不要だった——。繁茂の進み方はサイトごとに違い、経験則だけでは訪問計画が空振りします。

ISSUE — 03

影による売電損失は静かに進行する

縁辺部の樹木・雑草がパネルに影を落とすと、発電量は静かに削られます。「いつから・どこが」の記録がないと、対策の優先順位もつけられません。

What Sunscout Does

衛星の植生指標で、
「訪問すべきサイト」から順に並べ替える。

SUNSCOUT が提供するのは高精細な絵ではなく、意思決定に使える数値の時系列です。

月次NDVI監視

Sentinel-2 L2A の植生指標（NDVI）を、SCL バンドによる雲マスク付きで毎月算出。敷地全体の「緑の濃さ」を median 値で頑健に追跡します。

Sentinel-2 L2A / 10m / SCLマスク

前年同月比較

冬に枯れて夏に茂るのは正常な季節変動。前年同月との差分で見ることで、季節と本当の変化を切り分け、冬季の低NDVIを「異変」と誤読しない設計です。

YoY差分 / 季節変動と分離

訪問前提案ドラフト

観測値と積算根拠つきの提案文ドラフトを自動生成。損失試算は仮定をすべて開示し、金額は必ず「幅」で提示——確定は現地実測後、が原則です。

Honest Limits

できないことも、先にお伝えします。

衛星モニタリングは万能ではありません。限界を隠さないことが、使える判断材料の条件だと考えています。

パネル1枚の影は
見えませんRESOLUTION — 10m
Sentinel-2 の分解能は10m。パネル単位の影検出はできず、見えるのは「敷地の植生傾向」です。 だからこそ SUNSCOUT の役割は点検の代替ではなく、訪問の優先順位づけと、行く前の根拠づくりに絞っています。
曇天の月は
「欠測」になりますGAP — 8.3〜12.5%
日本の空は曇ります。有効画素が2割を切る月は補間でごまかさず「欠測」と明示します。 実測では24ヶ月中2〜3ヶ月が欠測でした。欠測率を最初から定量化して提示するのが SUNSCOUT の流儀です。

How It Works

公開データだけで、ここまで辿り着く。

特別なセンサーの設置は不要。国の公表情報と衛星オープンデータを組み合わせた4段パイプラインです。

発電所を特定

FIT事業計画認定情報（経産省公表）から対象発電所の所在地・認定出力を取得。

FIT公表情報

敷地輪郭を確定

ジオコーディングで概略位置を求め、航空写真から敷地ポリゴンをデジタイズ。面積とFIT出力の整合を検証。

国土地理院 API・地理院タイル

衛星で毎月観測

Sentinel-2 L2A を Statistical API で敷地集約。SCLバンドで雲・影・雪を除外し、NDVI median を算出。

レポート化
